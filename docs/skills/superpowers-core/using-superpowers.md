# 技能发现与使用入门指南

> 在任何对话开始前，确保正确发现和调用相关技能，避免遗漏关键工作流。

## 概述

`superpowers:using-superpowers` 是 Superpowers 插件的元技能（meta-skill），它定义了 Claude Code 如何发现、评估和调用技能的核心规则。该技能解决的核心问题是：**在处理用户请求之前，必须先检查是否有适用的技能**，即使只有 1% 的可能性也应该调用技能进行确认。

这个技能建立了一个严格的"先检查技能，再采取行动"的工作流程，确保 Claude Code 不会跳过任何可能有用的技能流程就直接开始工作。

### 核心机制

- **技能优先原则**：收到用户消息后，在做任何回复（包括澄清问题）之前，先检查是否有适用的技能
- **指令优先级体系**：用户显式指令 > Superpowers 技能 > 默认系统行为
- **平台适配**：支持 Claude Code（`Skill` 工具）、Copilot CLI（`skill` 工具）、Gemini CLI（`activate_skill` 工具）等多种平台

## 使用场景

- 每次对话开始时，用于建立技能发现和调用的基本规则
- 当不确定某个任务是否有对应技能可用时，用于指导判断
- 当需要理解多个技能之间的优先级和调用顺序时
- 当需要在不同平台上使用技能时，了解平台间的工具映射

## 用法

调用方式：`/superpowers:using-superpowers`

该技能通常在对话开始时自动生效，建立技能使用的基本规则。它不需要每次手动调用，但在以下情况下应手动调用：

- 开始一个全新的工作会话时
- 需要回顾技能使用规则时
- 在多技能冲突时需要确认优先级时

## 工作流程

技能定义了以下决策流程：

1. **收到用户消息** -- 判断是否有任何技能可能适用
2. **即使只有 1% 的可能性** -- 调用 `Skill` 工具加载对应技能
3. **宣告使用** -- 例如："使用 [技能名] 来 [目的]"
4. **检查清单** -- 如果技能包含清单，为每个条目创建 TodoWrite 任务
5. **严格遵循技能指引** -- 按照技能内容执行
6. **完成回复** -- 包括澄清问题在内的回复

### 技能优先级规则

当多个技能可能适用时，按以下顺序处理：

1. **流程类技能优先**（如 brainstorming、systematic-debugging）-- 决定如何处理任务
2. **实现类技能其次**（如 frontend-design、mcp-builder）-- 指导具体执行

### 技能类型

- **刚性技能**（如 TDD、debugging）：必须严格遵循，不得自行调整
- **柔性技能**（如 patterns）：可根据上下文灵活调整原则

## 示例

### 示例 1: 基本用法 -- 新功能开发

用户说："帮我添加一个用户登录功能"

正确流程：
1. 先调用 `superpowers:brainstorming` -- 因为这是创意/功能开发工作
2. 通过头脑风暴确定设计方案
3. 再调用 `superpowers:writing-plans` 编写实施计划
4. 最后通过 `superpowers:test-driven-development` 进行开发

### 示例 2: Bug 修复场景

用户说："这个测试一直失败，帮我看看"

正确流程：
1. 先调用 `superpowers:systematic-debugging` -- 因为这是调试场景
2. 按系统性调试流程找到根本原因
3. 再调用 `superpowers:test-driven-development` -- 编写回归测试并修复

### 示例 3: 识别合理化借口

以下想法都是需要警惕的"合理化借口"：

| 想法 | 实际情况 |
|------|----------|
| "这只是一个简单问题" | 问题也是任务，需要检查技能 |
| "让我先探索一下代码" | 技能会告诉你如何探索 |
| "我先收集一些信息" | 技能会告诉你如何收集信息 |
| "这不需要正式的技能" | 如果技能存在，就应该使用它 |
| "我记得这个技能的内容" | 技能会更新，应该读取当前版本 |
| "先做一件小事再说" | 做任何事之前先检查技能 |

## 注意事项

- **永远不要跳过技能检查**：即使任务看起来很简单，也要先检查是否有适用的技能
- **用户指令始终优先**：如果用户的 CLAUDE.md 或直接指令与技能冲突，以用户指令为准
- **技能内容会更新**：不要依赖记忆中的技能内容，每次都应通过 `Skill` 工具加载最新版本
- **与其他 superpowers 技能的关系**：本技能是所有其他 superpowers 技能的"入口"，它定义了发现和调用技能的基本规则，其他技能（如 brainstorming、writing-plans 等）是具体的工作流技能
- **平台差异**：不同平台（Claude Code、Copilot CLI、Gemini CLI）使用不同的工具名称来调用技能，但底层逻辑一致
